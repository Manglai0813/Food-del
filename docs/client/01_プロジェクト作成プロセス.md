# プロジェクト作成プロセス

## 概要
本プロジェクトは、既存のfood-del-clientを現代的なテクノロジースタックで再構築するプロジェクトです。

## プロジェクト初期化

### 1. Bunプロジェクトの作成
```bash
bun create vite food-del-client-v2 --template react-ts
cd food-del-client-v2
```

### 2. 基本設定ファイルの準備
- `package.json`: プロジェクトの依存関係とスクリプト定義
- `tsconfig.json`: TypeScript設定
- `vite.config.ts`: Viteビルドツール設定
- `.env`: 環境変数設定

### 3. 環境変数設定
```bash
# .env
VITE_API_URL=http://localhost:5000
```

## プロジェクト構造
```
food-del-client-v2/
├── src/
│   ├── components/        # 再利用可能なコンポーネント
│   ├── pages/            # ページコンポーネント
│   ├── hooks/            # カスタムフック
│   ├── services/         # API呼び出しサービス
│   ├── stores/           # Zustand状態管理
│   ├── types/            # TypeScript型定義
│   ├── utils/            # ユーティリティ関数
│   ├── styles/           # CSS Modulesスタイル
│   └── assets/           # 静的アセット
├── docs/                 # プロジェクト文書
└── public/              # パブリックアセット
```

## 特徴
- **Bun Runtime**: Node.jsの代わりにBunを使用
- **Vite**: 高速な開発サーバーとビルドツール
- **React 19**: 最新のReactバージョン
- **TypeScript**: 型安全性の確保
- **CSS Modules**: スタイルのカプセル化

## 設計原則
1. **型安全性**: TypeScriptによる厳格な型チェック
2. **コンポーネント分離**: 再利用可能なコンポーネント設計
3. **状態管理の明確化**: Zustandによる効率的な状態管理
4. **APIとの一貫性**: バックエンドAPIとの型定義同期
