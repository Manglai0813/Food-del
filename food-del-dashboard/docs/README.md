## 📝 ドキュメント貢献ガイドライン

### ドキュメント更新時のルール
1. **日本語で記述** - 全てのドキュメントは日本語で統一
2. **コード例を含める** - 実践的なコード例を必ず記載
3. **図表を活用** - ASCII図表で視覚的に説明
4. **バージョン管理** - 更新時は更新履歴を記録
5. **実践的な内容** - 理論だけでなく実装方法も記載

### ドキュメント追加時
```bash
# 新規ドキュメント作成
docs/09-新しいトピック.md

# README.mdに追加
# このファイルを更新して新しいドキュメントをリスト

# コミット
git add docs/
git commit -m "docs: 新しいトピックのドキュメントを追加"
```
