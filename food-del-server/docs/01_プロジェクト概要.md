# 01_プロジェクト概要

## フードデリバリーシステム - プロジェクト概要書

### 📋 プロジェクト基本情報

**プロジェクト名**: Food Delivery Server
**技術種別**: TypeScript Backend API
**開発環境**: Bun Runtime + Node.js エコシステム
**データベース**: PostgreSQL + Prisma ORM
**プロジェクト分類**: 学習プロジェクト（企業レベル実装）

---

### 🎯 プロジェクト目的

このプロジェクトは、現代的なフードデリバリーサービスのバックエンドAPIを構築し、以下の技術領域の習得を目指します：

1. **モダンバックエンド開発**: TypeScript + Bun + Express.js
2. **データベース設計**: 関係型データベースの正規化と最適化
3. **認証・認可システム**: JWT を使用したセキュアな認証機構
4. **在庫管理システム**: 楽観的ロックによる並行制御
5. **セキュリティ実装**: 多層セキュリティ防御機構
6. **エラーハンドリング**: 統合されたエラー処理システム

---

### 🚀 主要機能

#### **ユーザー管理機能**
- ユーザー登録・ログイン
- プロフィール管理

#### **注文管理機能**

#### **在庫管理機能**

### 🏗️ システムアーキテクチャ

#### **レイヤード・アーキテクチャ**
```
プレゼンテーション層 (Controllers)
    ↓
ビジネスロジック層 (Services)
    ↓
データアクセス層 (Prisma ORM)
    ↓
データベース層 (PostgreSQL)
```

#### **セキュリティ層**
- **認証ミドルウェア**: JWT 検証
- **レート制限**: API 保護
- **セキュリティヘッダー**: HTTP セキュリティ
- **入力検証**: データ整合性

### 🔄 開発フロー

#### **プロジェクト構造**
```
food-del-server/
├── src/
│   ├── controllers/     # HTTP リクエスト処理
│   ├── services/        # ビジネスロジック
│   ├── middleware/      # 横断的関心事
│   ├── routes/          # エンドポイント定義
│   ├── types/           # TypeScript 型定義
│   ├── lib/             # 共通ライブラリ
│   ├── utils/           # ヘルパー関数
│   └── errors/          # カスタムエラークラス
├── prisma/              # データベーススキーマ
├── docs/                # プロジェクト文書
└── package.json         # 依存関係とスクリプト
```
