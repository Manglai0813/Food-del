# 01. プロジェクト紹介とクイックスタート

## 1. はじめに

本文書は `food-del-server` プロジェクトの概要と、開発環境を迅速にセットアップするための手順を説明します。

このプロジェクトは、フードデリバリーアプリケーションのバックエンドAPIです。Node.js環境で動作し、TypeScriptを使用して開発されています。堅牢なアーキテクチャと最新のベストプラクティスに基づいて構築されており、高い保守性と拡張性を備えています。

## 2. 技術スタック

このプロジェクトで使用されている主要な技術は以下の通りです。

-   **ランタイム**: Node.js (Bunランタイムでの実行を推奨)
-   **言語**: TypeScript
-   **フレームワーク**: Express.js
-   **ORM**: Prisma
-   **データベース**: PostgreSQL
-   **主要なライブラリ**:
    -   `zod`: 型安全なデータバリデーション
    -   `jsonwebtoken` (JWT): 認証トークンの生成と検証
    -   `bcryptjs`: パスワードのハッシュ化
    -   `multer`: ファイルアップロード処理
    -   `helmet`: セキュリティ関連のHTTPヘッダー設定

## 3. 環境設定

プロジェクトを実行する前に、環境変数を設定する必要があります。

1.  プロジェクトのルートディレクトリに `.env` という名前のファイルを作成します。
2.  以下の内容をコピーし、実際の値に置き換えてください。

```env
# .env.example

# データベース接続URL (PostgreSQL)
# フォーマット: postgresql://USER:PASSWORD@HOST:PORT/DATABASE
DATABASE_URL="postgresql://your_user:your_password@localhost:5432/food_del_db"

# JWT署名用の秘密鍵 (32文字以上のランダムな文字列を強く推奨)
JWT_SECRET="your_super_secret_key_for_jwt_here"

# JWTリフレッシュトークン用の秘密鍵 (JWT_SECRETとは異なる値を設定)
JWT_REFRESH_SECRET="your_super_secret_key_for_refresh_token_here"

# サーバーがリッスンするポート (任意)
PORT=8000

# CORS設定 (フロントエンドのURLに合わせて変更)
CORS_ALLOWED_ORIGINS="http://localhost:3000,http://localhost:5173"
CORS_ALLOW_CREDENTIALS=true

# ... その他の任意設定
```

**重要**: `JWT_SECRET` と `JWT_REFRESH_SECRET` は、推測されにくいランダムな文字列を設定してください。

## 4. インストール

プロジェクトの依存関係をインストールします。ターミナルで以下のコマンドを実行してください。

```bash
npm install
```
*プロジェクトには `bun.lock` ファイルも含まれているため、Bunを使用している場合は `bun install` でもインストール可能です。*

## 5. 開発サーバーの起動

`package.json` に定義されているスクリプトを使用して、開発サーバーを起動します。

```bash
bun --watch index.ts
```

このコマンドはBunランタイムを使用し、ファイルの変更を監視して自動的にサーバーを再起動します（ホットリロード）。

サーバーが正常に起動すると、コンソールに `Server is running on port 8000` のようなメッセージが表示されます。
